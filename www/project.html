This Project

This website is the alternate Project 3 for the winter quarter of CSE P 561.

The requirement was to "Build an End-to-end personal website"

This site is accessible at https://lwxanalytics.com.

Notes

Version Control

  • All of the code for this site is available at https://github.com/Adam-Hoelscher/lwxanalytics.com
  • .gitignore was automatically generated by GitHub during the setup of the repo; few-if-not-no edits were needed
  • A symlink to the nginx site availability file was added for convenience; contents of the underlying file aren't visible in the repo

DNS

  • lwxanalytics.com was registered via Cloudflare
  • At the end there are two DNS records for this site
    • Direct the root domain to the server
      {"Type":"A", "Name":"lwxanalytics.com", "Content":"165.232.140.175"}
    • Direct www to the root domain
      {"Type":"CNAME", "Name":"www", "Content":"@"}
  • In the process of setting up the site some TXT records were created as part of the TLS certificate request process (see below)
  • Overall, this was fairly straight forward

TLS certificate

  • The certificate was obtained from Let's Encrypt
  • Let's Encrypt certificates are available via an automated process in GitHub pages. Unfortunately GitHub pages was not a viable way to host this page (see below)
  • The software certbot was used for the request
  • The way Let's Encrypt verifies ownership of a domain is elegant genius
    • LC gives the user a 'verification token'; the user adds a TXT record to DNS with that token in it
    • It's the same basic idea as TFA; by creating the record, the user proves they are in control of the domain
  • Originally, I requested a certificate for www.lwxanalytics.com
  • Later, I decided that I wanted the site to also be available at lwxanalytics.com. That lead me on a journey involving multiple requests, finally settling on a single cert that covers both

Hosting

  • GitHub pages doesn't allow the user to install their own TLS certificate
  • GitLab pages also doesn't allow user installed certificates
  • As a cheap and easy solution, this site is served on a Digital Ocean Droplet
    • 'Droplet' is cute branding; it's a VM
    • The server seems to be weaker than a G1 Raspberry Pi; I started the content development running VS Code via SSH; after the 3rd time it locked up for over an hour, I switched tact (see below)
  • The Droplet started with nothing but the OS on it; updating all the installed software and installing nginx was sufficient to allow it act as a server

Content

  • The first version of the content was created with ChatGPT; I fed it my PMP application and asked it make me a site with a couple pages
  • I applied some lite editing to some of the language and then made some more significant changes
    • I moved the nav bar to its own source file and added JS to the pages to dynamically load it; DRY
    • I added this page
  • After the Droplet's lock-ups I switched to creating content in the repo using my local machine; updates are done by pushing to GitHub from local and pulling from the Droplet
